## 核心模块说明

### 1. 后端API (backend/)

**路由模块:**
- `auth.js` - 用户注册、登录、认证
- `data.js` - 数据上传、下载、管理
- `review.js` - 审核流程管理
- `admin.js` - 系统管理功能
- `users.js` - 用户信息管理
- `ai.js` - AI检测接口

**中间件:**
- `auth.js` - JWT认证、角色授权
- `upload.js` - 文件上传处理
- `audit.js` - 操作日志记录

### 2. 前端应用 (frontend/)

**页面视图:**
- **认证页面**: Login.vue, Register.vue
- **仪表盘**: Dashboard.vue - 数据概览
- **数据管理**: data/Upload.vue, List.vue, Detail.vue
- **审核中心**: review/Pending.vue, History.vue
- **管理后台**: admin/Dashboard.vue, Users.vue, Data.vue, Logs.vue
- **个人中心**: Profile.vue, Notifications.vue

### 3. AI服务 (ai-service/)

**核心功能:**
- 数据文件解析 (CSV, Excel, JSON)
- 数据质量检测 (缺失值、重复数据、异常值)
- 统计分析 (均值、标准差、分布)
- 异常检测 (极端值、数据不一致)
- 质量评分 (0-100分)

### 4. 数据库 (mysql/)

**核心表:**
- `users` - 用户信息
- `data_submissions` - 数据提交
- `review_records` - 审核记录
- `teacher_student_relations` - 导师-学生关系
- `notifications` - 通知消息
- `quota_usage_logs` - 额度使用记录
- `access_logs` - 访问日志

## 数据流

```
用户上传数据
    ↓
文件存储 + 计算哈希
    ↓
数据记录创建 (状态: draft)
    ↓
AI自动检测 (异步)
    ↓
用户提交审核
    ↓
导师一审 (teacher_reviewing)
    ↓
专家盲审 (expert_reviewing)
    ↓
管理员终审 (final_approved/rejected)
    ↓
通知用户结果
```

## 配置文件

### 环境变量 (.env)

```
# 数据库
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=VeriData@2024
DB_NAME=veri_data

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# JWT
JWT_SECRET=your-secret-key
JWT_EXPIRES_IN=7d

# 服务
PORT=3000
NODE_ENV=development

# AI服务
AI_SERVICE_URL=http://localhost:5000
```

### Docker配置

```yaml
# 服务
- frontend: Nginx (端口 80)
- backend: Node.js (端口 3000)
- ai-service: Python Flask (端口 5000)
- mysql: MySQL 8.0 (端口 3306)
- redis: Redis 7 (端口 6379)

# 数据卷
- mysql/data: 数据库持久化
- uploads: 上传文件
- logs: 日志文件
```

## 开发指南

### 启动开发环境

```bash
# 完整启动
docker-compose up -d

# 只启动基础设施
docker-compose up -d mysql redis

# 后端开发
cd backend
npm install
npm run dev

# 前端开发
cd frontend
npm install
npm run serve

# AI服务开发
cd ai-service
pip install -r requirements.txt
python app.py
```

### 调试技巧

```bash
# 查看日志
docker-compose logs -f backend
docker-compose logs -f ai-service

# 进入容器
docker-compose exec backend sh
docker-compose exec mysql mysql -u root -p

# 重启服务
docker-compose restart backend
```
