# 鉴真数据系统 - 部署指南

## 快速启动（推荐）

### Windows 系统

1. 确保 Docker Desktop 已启动
2. 打开 PowerShell，进入项目目录：
```powershell

```

3. 运行启动脚本：
```powershell

```

### Linux/Mac 系统

```bash

chmod +x start.sh
./start.sh
```

## 常用操作

### 查看日志

```bash
# 所有服务
docker-compose logs -f

# 特定服务
docker-compose logs -f backend
docker-compose logs -f ai-service
docker-compose logs -f frontend
docker-compose logs -f mysql
```

### 重启服务

```bash
# 重启特定服务
docker-compose restart backend

# 重启所有服务
docker-compose restart
```

### 停止服务

```bash
# 停止但保留数据
docker-compose down

# 停止并删除所有数据（慎用）
docker-compose down -v
```

### 进入容器调试

```bash
# 后端
docker-compose exec backend sh

# MySQL
docker-compose exec mysql mysql -u root -p

# Redis
docker-compose exec redis redis-cli
```

## 故障排查

### 端口冲突

如果端口被占用，修改 `docker-compose.yml` 中的端口映射：

```yaml
ports:
  - "3307:3306"  # 改为 3307
```

### 数据库连接失败

1. 检查 MySQL 是否启动：
```bash
docker-compose ps mysql
```

2. 检查数据库初始化：
```bash
docker-compose logs mysql | grep "init"
```

3. 手动执行初始化脚本：
```bash
docker-compose exec -T mysql mysql -u root -pVeriData@2024 veri_data < mysql/init/01-schema.sql
```

### AI 服务连接失败

检查后端环境变量中的 `AI_SERVICE_URL` 配置：
```bash
docker-compose logs backend | grep "AI"
```

### 前端无法访问 API

检查 `frontend/nginx.conf` 中的代理配置：
```nginx
location /api {
    proxy_pass http://backend:3000;
}
```

## 生产环境部署

### 1. 修改环境变量

创建生产环境配置：

```bash
# backend/.env.production
NODE_ENV=production
DB_HOST=your-mysql-host
DB_PASSWORD=your-secure-password
JWT_SECRET=your-random-secret-key
```

### 2. 配置 HTTPS

修改 `frontend/nginx.conf`：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # ... 其他配置
}
```

### 3. 使用 Docker Swarm

```bash
# 初始化 Swarm
docker swarm init

# 部署服务
docker stack deploy -c docker-compose.yml veri-data
```

### 4. 数据备份

```bash
# 备份 MySQL
docker-compose exec mysql mysqldump -u root -p veri_data > backup_$(date +%Y%m%d).sql

# 备份上传文件
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz uploads/
```

## 性能优化

### 1. 数据库优化

```sql
-- 添加常用查询索引
CREATE INDEX idx_data_status ON data_submissions(review_status);
CREATE INDEX idx_data_submitter ON data_submissions(submitter_id);
CREATE INDEX idx_review_status ON review_records(status);
```

### 2. 缓存优化

在 `backend/.env` 中启用 Redis 缓存：
```
REDIS_HOST=redis
REDIS_PORT=6379
```

### 3. 文件存储优化

大文件建议使用对象存储（如 MinIO）：
```yaml
services:
  minio:
    image: minio/minio
    command: server /data
```

## 安全建议

1. **修改默认密码**：立即修改所有默认密码
2. **启用防火墙**：限制数据库和 Redis 端口访问
3. **定期备份**：设置自动备份任务
4. **日志审计**：定期检查访问日志
5. **更新依赖**：定期更新 Docker 镜像和依赖包
